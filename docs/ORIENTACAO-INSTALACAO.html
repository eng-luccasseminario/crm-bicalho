Orientação — Instalar o CRM Bicalho

Pontapé para colocar o CRM no ar do zero. O guia técnico completo é o docs/SETUP.md do repositório.

Repositório (público): github.com/eng-luccasseminario/crm-bicalho

1. Antes de tudo

Instale na sua máquina

  • Node.js 18+ — nodejs.org
  • Git — git-scm.com
  • Railway CLI — npm i -g @railway/cli

Crie estas contas

Todas com free tier, exceto a OpenAI (paga por uso):

SupabaseBanco de dados do CRMGrátis

2. Clonar e seguir o guia

O repositório é público: não precisa de login para clonar.

git clone https://github.com/eng-luccasseminario/crm-bicalho.git
cd crm-bicalho/agente-whatsapp
npm install

Depois, siga o docs/SETUP.md de cima a baixo — ele tem um checklist no topo, os links de cada credencial, um conector automático (npm run conectar) e o troubleshooting.

3. Os 5 alertas que evitam 90% dos problemas

1Supabase — ao copiar a connection string, troque [YOUR-PASSWORD] pela senha real do banco. Deixe o Pool Size em 30.

2Google (o mais importante) — na tela de consentimento OAuth, PUBLIQUE em "Production" (não deixe em "Testing"). Em Testing o token expira em 7 dias e o sistema para de gravar no Drive toda semana. Aceite o aviso de "app não verificado".

3Twenty no Railway — depois do serviço web, crie o serviço worker (2º serviço, start command yarn worker:prod). Sem ele a Timeline não funciona e as tarefas não marcam. Em ambos adicione PG_POOL_IDLE_TIMEOUT_MS=10000 e PG_POOL_ALLOW_EXIT_ON_IDLE=true.

4Agente — tem projeto próprio no Railway (separado do Twenty). Crie com railway up --new -d de dentro de agente-whatsapp.

5Conectar tudo — de dentro de agente-whatsapp, rode npm run conectar: pergunta cada chave, valida na hora e abre o OAuth do Google.

4. Finalização

  • npx ts-node scripts/criar-campo-pasta-drive.ts — cria o campo "Pasta no Drive" na Empresa.
  • npm run sync:crm — traz documentos antigos do Drive para o CRM.
  • Testes de fumaça (§9 do SETUP): manda mensagem pro bot, envia um PDF, cadastra uma proposta.

5. Documentos de apoio no repositório

  • docs/SETUP.md — guia técnico completo (o principal).
  • docs/PLATAFORMAS.md — tabela de plataformas e onde pegar cada credencial.
  • docs/ATIVAR-WHATSAPP.md — migrar do Telegram para o WhatsApp oficial.
